---
title: 'CameraDevice'
description: 'Types representing a physical or logical camera device and its supported formats.'
---

A `CameraDevice` is returned by `Camera.getAvailableCameraDevices()` and describes a camera the system has discovered. A single logical device may combine multiple physical cameras (e.g. a Triple Camera).

```ts
import { Camera } from 'react-native-vision-camera'

const devices = await Camera.getAvailableCameraDevices()
const back = devices.find((d) => d.position === 'back')
```

---

## CameraPosition

The physical location of a camera on the device.

```ts
type CameraPosition = 'front' | 'back' | 'external'
```

| Value | Description |
|---|---|
| `'back'` | Located on the back of the device. Used for capturing the environment in front of the user. |
| `'front'` | Located on the front of the device. Used for selfies and video calls. |
| `'external'` | An external camera with no fixed facing (e.g. USB, Continuity Camera, Bluetooth/WiFi). |

---

## PhysicalCameraDeviceType

Identifies the type of a physical lens element. Multi-camera logical devices list all constituent physical types in `physicalDevices`.

```ts
type PhysicalCameraDeviceType =
  | 'ultra-wide-angle-camera'
  | 'wide-angle-camera'
  | 'telephoto-camera'
```

| Value | Field of view |
|---|---|
| `'ultra-wide-angle-camera'` | 94° or higher |
| `'wide-angle-camera'` | Between 60° and 94° |
| `'telephoto-camera'` | 60° or lower |

Common logical combinations:

- `ultra-wide-angle-camera` + `wide-angle-camera` → dual wide-angle camera
- `wide-angle-camera` + `telephoto-camera` → dual camera
- All three → triple camera

## AutoFocusSystem

The autofocus mechanism used by a `CameraDeviceFormat`.

```ts
type AutoFocusSystem = 'contrast-detection' | 'phase-detection' | 'none'
```

| Value | Description |
|---|---|
| `'none'` | Autofocus is not available. |
| `'contrast-detection'` | Focus achieved by scanning for peak contrast. More visually intrusive. |
| `'phase-detection'` | Focus achieved by phase comparison. Often achieves focus without a full scan. |

---

## VideoStabilizationMode

Controls electronic or optical video stabilization. Enabling stabilization may add latency to the capture pipeline.

```ts
type VideoStabilizationMode =
  | 'off'
  | 'standard'
  | 'cinematic'
  | 'cinematic-extended'
  | 'auto'
```

| Value | Description |
|---|---|
| `'off'` | No stabilization. |
| `'standard'` | Software stabilization. Reduces field of view by ~10%. |
| `'cinematic'` | More aggressive software stabilization with additional cropping. |
| `'cinematic-extended'` | Combined software and hardware stabilization with the most aggressive crop. |
| `'auto'` | Platform selects the most appropriate mode automatically. |

## CameraDeviceFormat

A single stream configuration for a camera device, describing resolution, frame-rate range, and capabilities. Use `device.formats` to find the best format for your use case and pass it to `<Camera format={format} />`.

<ResponseField name="photoWidth" type="number" required>

</ResponseField>

<ResponseField name="videoHeight" type="number" required>
  The height of the video stream resolution, in pixels.
</ResponseField>

<ResponseField name="videoWidth" type="number" required>
  The width of the video stream resolution, in pixels.
</ResponseField>

<ResponseField name="maxISO" type="number" required>
  Maximum supported ISO value for this format.
</ResponseField>

<ResponseField name="minISO" type="number" required>
  Minimum supported ISO value for this format.
</ResponseField>

<ResponseField name="fieldOfView" type="number" required>
  The horizontal field of view of the video stream, in degrees.
</ResponseField>

<ResponseField name="supportsVideoHdr" type="boolean" required>
  Whether this format supports HDR for video capture.
</ResponseField>

<ResponseField name="supportsPhotoHdr" type="boolean" required>
  Whether this format supports HDR for photo capture.
</ResponseField>

<ResponseField name="supportsDepthCapture" type="boolean" required>
  Whether this format can deliver depth data for photo or video capture.
</ResponseField>

<ResponseField name="minFps" type="number" required>

## CameraDevice

Represents a camera device discovered by `Camera.getAvailableCameraDevices()`.

<ResponseField name="id" type="string" required>
  The native platform identifier for this camera device instance. Use this to persist a device selection across app launches.
</ResponseField>

<ResponseField name="physicalDevices" type="PhysicalCameraDeviceType[]" required>
  The physical lenses this logical device combines.

  - Single-element arrays indicate a physical camera (e.g. `['wide-angle-camera']`).
  - Multi-element arrays indicate a logical camera (e.g. `['wide-angle-camera', 'telephoto-camera']`).

  Use `isMultiCam` to check at a glance whether this is a multi-camera device.
</ResponseField>

<ResponseField name="position" type="CameraPosition" required>
  The physical mounting position of this camera. One of `'front'`, `'back'`, or `'external'`.
</ResponseField>

<ResponseField name="name" type="string" required>
  A localized, human-readable name for this camera (e.g. `"Back Triple Camera"`).
</ResponseField>

<ResponseField name="hasFlash" type="boolean" required>
  Whether this camera supports enabling the flash for photo capture.
</ResponseField>

<ResponseField name="hasTorch" type="boolean" required>
  Whether this camera supports continuously enabling the flash as a torch (i.e. flash during video capture).
</ResponseField>

<ResponseField name="minFocusDistance" type="number" required>
  The minimum distance this device can properly focus to, in centimeters. Returns `0` if unknown.
</ResponseField>

<ResponseField name="isMultiCam" type="boolean" required>
  Whether this is a virtual multi-camera that combines multiple physical cameras. Multi-cameras can seamlessly switch between lenses while zooming and may produce depth data.
</ResponseField>

<ResponseField name="minZoom" type="number" required>
  The minimum zoom factor available (e.g. `1`).
</ResponseField>

<ResponseField name="maxZoom" type="number" required>
  The maximum zoom factor available (e.g. `128`).
</ResponseField>

<ResponseField name="neutralZoom" type="number" required>
  The zoom factor at which the camera is in its "natural" wide-angle mode — before it switches to ultra-wide or telephoto.

  - For single physical cameras this is always `1.0`.
  - For multi-cameras this is a value between `minZoom` and `maxZoom`.

  Use this as the initial value for a custom zoom control so you don't start in ultra-wide mode.

  ```ts
  const zoom = useSharedValue(device.neutralZoom)
  ```
</ResponseField>

<ResponseField name="minExposure" type="number" required>
  The minimum exposure bias value supported. At this value the image is nearly completely dark (under-exposed).
</ResponseField>

<ResponseField name="maxExposure" type="number" required>
  The maximum exposure bias value supported. At this value the image is nearly completely bright (over-exposed).
</ResponseField>

<ResponseField name="supportsLowLightBoost" type="boolean" required>
  Whether this device supports the low-light boost feature.
</ResponseField>

<ResponseField name="supportsRawCapture" type="boolean" required>
  Whether this device supports capturing photos in RAW format.

  <Warning>RAW capture support is a work in progress.</Warning>
</ResponseField>

<ResponseField name="supportsFocus" type="boolean" required>
  Whether this device supports tap-to-focus via `Camera.focus(...)`.
</ResponseField>

<ResponseField name="hardwareLevel" type="'legacy' | 'limited' | 'full'" required>
  The hardware capability level of this camera.

  - On Android, older devices may report `'legacy'` or `'limited'`, indicating backwards-compatible operation.
  - On iOS, all devices report `'full'`.
</ResponseField>

<ResponseField name="sensorOrientation" type="Orientation" required>
  The physical orientation of the camera sensor relative to the phone. Camera sensors are typically mounted landscape, so this is commonly `'landscape-left'` or `'landscape-right'`.

  All camera streams are delivered in this orientation. Consumers must counter-rotate by this amount to produce an upright image:

  - **Photo/video**: handled automatically via EXIF flags.
  - **Preview**: handled via a view transform matrix.
  - **Frame processors**: must be handled manually using `Frame.orientation`.
</ResponseField>

<ResponseField name="formats" type="CameraDeviceFormat[]" required>
  All stream configurations supported by this device. Inspect these to find the best format for your resolution, frame rate, and feature requirements, then pass the result to `<Camera format={format} />`.

  <Expandable title="CameraDeviceFormat properties">
    <ResponseField name="photoHeight" type="number" required>
      Maximum photo capture height, in pixels.
    </ResponseField>
    <ResponseField name="photoWidth" type="number" required>
      Maximum photo capture width, in pixels.
    </ResponseField>
    <ResponseField name="videoHeight" type="number" required>
      Video stream height, in pixels.
    </ResponseField>
    <ResponseField name="videoWidth" type="number" required>
      Video stream width, in pixels.
    </ResponseField>
    <ResponseField name="maxISO" type="number" required>
      Maximum supported ISO.
    </ResponseField>
    <ResponseField name="minISO" type="number" required>
      Minimum supported ISO.
    </ResponseField>
    <ResponseField name="fieldOfView" type="number" required>
      Horizontal field of view in degrees.
    </ResponseField>
    <ResponseField name="supportsVideoHdr" type="boolean" required>
      Whether HDR video capture is supported.
    </ResponseField>
    <ResponseField name="supportsPhotoHdr" type="boolean" required>
      Whether HDR photo capture is supported.
    </ResponseField>
    <ResponseField name="supportsDepthCapture" type="boolean" required>
      Whether depth data delivery is supported.
    </ResponseField>
    <ResponseField name="minFps" type="number" required>
      Minimum achievable frame rate for this format.
    </ResponseField>
    <ResponseField name="maxFps" type="number" required>
      Maximum achievable frame rate for this format.
    </ResponseField>
    <ResponseField name="autoFocusSystem" type="AutoFocusSystem" required>
      The autofocus mechanism: `'contrast-detection'`, `'phase-detection'`, or `'none'`.
    </ResponseField>
    <ResponseField name="videoStabilizationModes" type="VideoStabilizationMode[]" required>
      Supported video stabilization modes for this format.
    </ResponseField>
  </Expandable>
</ResponseField>
