Frame Processor Plugins are native plugins that process Camera frames and return values to your JavaScript Frame Processor. They are written in native languages such as Java/Kotlin, Objective-C/Swift, or C++ to take advantage of GPU acceleration, low-level APIs, and high performance.

Like React Native libraries, Frame Processor Plugins are distributed through npm and support autolinking.

## Installing a plugin

<Steps>
  <Step title="Install the package">
    ```bash
    npm i vision-camera-xxxxx
    ```

    For iOS, also run:

    ```bash
    cd ios && pod install
    ```
  </Step>
  <Step title="Rebuild your app">
    Native plugins require a full rebuild to autolink correctly.
  </Step>
  <Step title="Use the plugin in your Frame Processor">
    ```tsx
    import { useFaceDetector } from 'vision-camera-face-detector'

    function App() {
      const { detectFaces } = useFaceDetector()

      const frameProcessor = useFrameProcessor((frame) => {
        'worklet'
        const faces = detectFaces(frame)
        console.log(`Detected ${faces.length} face(s)`)
      }, [detectFaces])

      return <Camera frameProcessor={frameProcessor} {...props} />
    }
    ```
  </Step>
</Steps>

## Finding more plugins

Search npm for [`vision-camera`](https://www.npmjs.com/search?q=vision-camera) to find additional community plugins. Plugins typically follow the naming convention `vision-camera-*` or `react-native-vision-camera-*`.

To add your own plugin to the official VisionCamera plugin list, open a pull request to [edit the community plugins page](https://github.com/mrousavy/react-native-vision-camera/edit/main/docs/docs/guides/FRAME_PROCESSOR_PLUGINS_COMMUNITY.mdx) in the VisionCamera repository.
