VisionCamera surfaces errors through a typed system so you can identify the exact cause and respond appropriately. Every error has a `code` in the form `domain/code`, a human-readable `message`, and an optional `cause` with native details.

## Error types

### CameraRuntimeError

`CameraRuntimeError` represents errors that occur in the camera session itself â€” in the preview, outputs, or device configuration. The camera surfaces these through the `onError` prop on `<Camera>`.

```tsx
import { Camera, CameraRuntimeError } from 'react-native-vision-camera'
import { useCallback } from 'react'

function App() {
  const onError = useCallback((error: CameraRuntimeError) => {
    console.error(`Camera error [${error.code}]: ${error.message}`)
  }, [])

  return <Camera onError={onError} {...cameraProps} />
}
```

`CameraRuntimeError` covers codes from the `permission/`, `parameter/`, `device/`, `format/`, `session/`, `system/`, and `unknown/` domains.

### CameraCaptureError

`CameraCaptureError` represents errors that occur during photo capture or video recording. These are thrown as Promise rejections from `takePhoto`, `takeSnapshot`, `startRecording`, `stopRecording`, `pauseRecording`, `resumeRecording`, and `cancelRecording`.

```tsx
import { Camera, CameraCaptureError } from 'react-native-vision-camera'
import { useCallback, useRef } from 'react'

function App() {
  const camera = useRef<Camera>(null)

  const onPressCapture = useCallback(async () => {
    try {
      const photo = await camera.current?.takePhoto()
      console.log('Photo saved to:', photo?.path)
    } catch (e) {
      if (e instanceof CameraCaptureError) {
        switch (e.code) {
          case 'capture/file-io-error':
            console.error('Failed to write photo to disk.')
            break
          case 'capture/insufficient-storage':
            console.error('Not enough storage space.')
            break
          default:
            console.error(e)
            break
        }
      }
    }
  }, [camera])

  return <Camera ref={camera} {...cameraProps} />
}
```

## Error properties

Every error (both `CameraRuntimeError` and `CameraCaptureError`) extends `CameraError` and exposes:

| Property | Type | Description |
|---|---|---|
| `code` | `string` | Typed code in the form `domain/code`. |
| `message` | `string` | Non-localized description of the error. |
| `cause` | `Error \| undefined` | The underlying native error, if available. |

The `cause` wraps a native `NSError` (iOS) or `Throwable` (Android) with additional fields:

| Field | Platform | Description |
|---|---|---|
| `cause.message` | Both | Native error message. |
| `cause.code` | iOS only | `NSError.code`. |
| `cause.domain` | iOS only | `NSError.domain`. |
| `cause.details` | iOS only | `NSError.userInfo` dictionary. |
| `cause.stacktrace` | Android only | Java `Throwable` stack trace string. |
| `cause.cause` | Both | Recursive cause chain. |

## Switching on error codes

Use a `switch` statement on `error.code` to respond to specific conditions:

```ts
import { CameraRuntimeError } from 'react-native-vision-camera'

function handleCameraError(error: CameraRuntimeError) {
  switch (error.code) {
    case 'device/configuration-error':
      // Try a different format or reduce enabled outputs
      break
    case 'device/microphone-unavailable':
      // Disable audio and retry
      break
    case 'session/audio-in-use-by-other-app':
      // Notify user that another app is using audio
      break
    case 'permission/camera-permission-denied':
      // Open app settings
      break
    default:
      console.error(error)
      break
  }
}
```

## Full error handling example

```tsx
import React, { useCallback, useRef } from 'react'
import { Camera, CameraRuntimeError, CameraCaptureError } from 'react-native-vision-camera'
import type { CameraProps } from 'react-native-vision-camera'

function CameraScreen() {
  const camera = useRef<Camera>(null)

  // Handle session-level runtime errors via the onError prop
  const onError = useCallback((error: CameraRuntimeError) => {
    switch (error.code) {
      case 'permission/camera-permission-denied':
        alert('Camera permission was denied. Please enable it in Settings.')
        break
      case 'device/camera-already-in-use':
        alert('The camera is in use by another app.')
        break
      case 'system/camera-module-not-found':
        alert('VisionCamera native module is not linked correctly.')
        break
      default:
        console.error(`[${error.code}]: ${error.message}`, error.cause)
        break
    }
  }, [])

  // Handle capture errors via try/catch
  const takePhoto = useCallback(async () => {
    try {
      const photo = await camera.current?.takePhoto({ flash: 'auto' })
      console.log('Captured:', photo?.path)
    } catch (e) {
      if (e instanceof CameraCaptureError) {
        switch (e.code) {
          case 'capture/file-io-error':
            alert('Failed to save the photo. Check storage permissions.')
            break
          case 'capture/insufficient-storage':
            alert('Your device is out of storage.')
            break
          default:
            console.error(e)
            break
        }
      }
    }
  }, [camera])

  return <Camera ref={camera} onError={onError} {...cameraProps} />
}
```

<Note>
  If you do not pass an `onError` handler to `<Camera>`, runtime errors are automatically logged to the console via `console.error`. Always handle errors in production apps.
</Note>
